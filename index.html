Introducción

Fundamentos del análisis de vulnerabilidades en ciberseguridad

¿Qué es una vulnerabilidad?

Una vulnerabilidad es una debilidad o fallo en un sistema, aplicación, proceso o control de seguridad que puede ser explotado por una amenaza para comprometer la confidencialidad, integridad o disponibilidad (CIA) de la información.

Amenaza vs Vulnerabilidad vs Riesgo

  • Amenaza: Evento o agente con potencial de causar daño (ej. hacker, malware).
  • Vulnerabilidad: Debilidad que puede ser aprovechada.
  • Riesgo: Probabilidad e impacto de que una amenaza explote una vulnerabilidad.

Impacto en la seguridad

Las vulnerabilidades no parcheadas permiten accesos no autorizados, robo de datos, interrupción de servicios y daño reputacional. Su gestión proactiva es esencial en cualquier SOC.

CVE – Common Vulnerabilities and Exposures

Sistema estandarizado de identificación de vulnerabilidades

Definición: CVE es un diccionario público de vulnerabilidades de seguridad informática con identificadores únicos asignados por MITRE y mantenidos en coordinación con la comunidad global.

Estructura de un CVE

CVE -2021- 44228
  • Prefijo: "CVE" identifica el estándar
  • Año: Año de asignación o publicación
  • ID: Número secuencial único (4+ dígitos)

Cómo consultar vulnerabilidades

  • cve.mitre.org
  • nvd.nist.gov (NVD)
  • cvedetails.com

CVSS – Common Vulnerability Scoring System

Sistema de puntuación para evaluar la severidad de vulnerabilidades

Definición

CVSS proporciona una forma abierta y estandarizada de capturar las características principales de una vulnerabilidad y producir un puntaje numérico (0.0 – 10.0) que refleja su severidad.

Escala de severidad

0.0Ninguno
0.1–3.9Bajo
4.0–6.9Medio
7.0–8.9Alto
9.0–10.0Crítico

Pasa el cursor sobre cada segmento para ver detalles.

Métricas CVSS v3.1

Tipo de Score Descripción Uso Acción

Temporal Score Modifica el Base según factores temporales Exploits disponibles, parches
Environmental Score Personalizado al entorno organizacional Impacto en activos específicos

ExploitDB

Base de datos pública de exploits y pruebas de concepto

¿Qué es ExploitDB?

Exploit Database (ExploitDB) es un repositorio mantenido por Offensive Security que cataloga exploits públicos, shellcodes y papers de investigación de seguridad asociados a vulnerabilidades conocidas.

Relación CVE ↔ Exploit

Un CVE identifica la vulnerabilidad; un exploit en ExploitDB es código o técnica que demuestra cómo explotarla. No todo CVE tiene exploit público disponible.

Uso ético

Solo utilizar en entornos autorizados, laboratorios controlados o con permiso explícito. El uso no autorizado es ilegal y antiético.

Flujo: Vulnerabilidad → Riesgo

Vulnerability Scanning

Proceso automatizado de detección de debilidades

Concepto

El escaneo de vulnerabilidades es el proceso de identificar, analizar y reportar debilidades de seguridad en sistemas, redes y aplicaciones mediante herramientas especializadas.

Diagrama de proceso

1

Definir alcance

2

Configurar escaneo

3

Ejecutar scan

4

Analizar resultados

5

Remediar

Ventajas

  • Detección automatizada y rápida
  • Cobertura de múltiples activos
  • Priorización basada en CVSS
  • Cumplimiento normativo (PCI-DSS, ISO 27001)

Limitaciones

  • No detecta lógica de negocio vulnerable
  • Puede generar falsos positivos/negativos
  • Requiere validación manual
  • Impacto en rendimiento de sistemas

Falsos positivos

El escaneo reporta una vulnerabilidad que en realidad no existe o no es explotable en el contexto del activo. Requiere verificación manual.

Falsos negativos

El escaneo no detecta una vulnerabilidad real presente en el sistema. Puede ocurrir por firmas desactualizadas o configuraciones personalizadas.

OpenVAS

Escáner de vulnerabilidades open source

Descripción

OpenVAS (Open Vulnerability Assessment System) es una solución completa de escaneo de vulnerabilidades de código abierto, integrada en Greenbone Vulnerability Management.

Arquitectura

  • Greenbone Management: Interfaz web y gestión
  • OpenVAS Scanner: Motor de escaneo
  • Feed NVT: Base de pruebas actualizada

Ventajas

  • Gratuito y open source
  • Amplia base de NVTs
  • Reportes detallados

Casos de uso

  • Auditorías de seguridad en PYMEs
  • Escaneos periódicos de infraestructura
  • Laboratorios educativos de ciberseguridad
  • Evaluación de cumplimiento normativo

Nessus

Escáner comercial líder en la industria

Descripción

Nessus, desarrollado por Tenable, es uno de los escáneres de vulnerabilidades más utilizados en la industria, conocido por su precisión y extensa base de plugins.

Características

  • Escaneos credentialed y non-credentialed
  • Integración con SIEM y ticketing
  • Reportes ejecutivos y técnicos

Nikto

Escáner de vulnerabilidades para servidores web

¿Qué es Nikto?

Nikto es una herramienta open source de escaneo de servidores web que detecta archivos peligrosos, configuraciones incorrectas, versiones obsoletas y más de 6,700 checks conocidos.

Principales funciones

  • Detección de CGI peligrosos
  • Identificación de versiones de software
  • Búsqueda de archivos y directorios sensibles
  • Detección de headers inseguros

Casos de uso

  • Auditoría rápida de servidores web
  • Reconocimiento en pentesting web
  • Verificación post-despliegue

Ejemplos de comandos

Escaneo por IP
nikto -h 192.168.1.10
Escaneo HTTPS
nikto -h https://empresa.com

III

Laboratorio Práctico

Escenario

Se dispone de una máquina virtual Metasploitable accesible desde Kali Linux.

Se recomienda utilizar la máquina Metasploitable2, disponible en VulnHub: vulnhub.com/entry/metasploitable-2,29.

Topología de laboratorio

Kali Linux Atacante / Escáner
Red NAT / Host-Only
Metasploitable 2 Objetivo vulnerable

Requisitos

  • VirtualBox o VMware instalado
  • VM Kali Linux actualizada
  • VM Metasploitable 2
  • Red configurada (NAT o Host-Only)
  • Autorización del instructor para el laboratorio

Procedimiento paso a paso

  1. Paso 1 – Identificar la dirección IP de Metasploitable
    Metasploitable
    ip addr

    o

    Alternativa
    ifconfig
  2. Paso 2 – Verificar conectividad
    Kali Linux
    ping <IP_Metasploitable>
  3. Paso 3 – Escaneo de puertos y vulnerabilidades con Nmap

    Desde Kali Linux, utilice Nmap para identificar puertos abiertos, servicios, sistema operativo y posibles vulnerabilidades del objetivo antes de pasar a un escáner dedicado.

    Escaneo SYN de puertos (host único)
    sudo nmap -v --reason -sS -oX puertos.xml --stylesheet="https://svn.nmap.org/nmap/docs/nmap.xsl" 192.168.141.134
    Detección de versión y SO (red completa)
    sudo nmap -v --reason -sV -O -oX resultados.xml --stylesheet="https://svn.nmap.org/nmap/docs/nmap.xsl" 192.168.141.0/24
    Detección de vulnerabilidades (scripts NSE)
    sudo nmap -sV --script vuln 192.168.141.134
    • -sS: escaneo SYN (sigiloso) para detectar puertos abiertos.
    • -sV: identifica la versión de los servicios encontrados.
    • -O: intenta determinar el sistema operativo del objetivo.
    • -oX y --stylesheet: exportan el resultado a XML y lo presentan como reporte HTML navegable.
    • --script vuln: ejecuta scripts NSE que cruzan los servicios detectados contra vulnerabilidades conocidas.

    Guarde los archivos puertos.xml y resultados.xml, junto con la salida del escaneo de vulnerabilidades, como evidencia del laboratorio.

  4. Paso 4 – Ejecutar Nikto
    Escaneo web
    nikto -h http://<IP_Metasploitable>

    Guarde los resultados obtenidos como evidencia.

  5. Paso 5 – Ejecutar OpenVAS o Nessus

    Realice un escaneo completo de vulnerabilidades contra la IP de Metasploitable.

  6. Paso 6 – Analizar resultados

    Identifique y documente:

    • Vulnerabilidades críticas
    • Vulnerabilidades altas
    • Vulnerabilidades medias
    • Servicios afectados
